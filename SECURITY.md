# Security Policy

## Reporting a vulnerability

Do not publish credentials, access tokens, or exploitable details in a public issue.
Report the problem privately to the repository owner through GitHub's private vulnerability reporting feature when it is enabled.

Include the affected image, Dockerfile path, reproduction steps, expected impact, and any suggested remediation.

## Supported images

Only the image tags produced from the current `main` branch are maintained:

- `k-ubuntu`
- `r-ubuntu`
- `t-ubuntu`

The images are rebuilt weekly to pick up current Ubuntu security updates.
